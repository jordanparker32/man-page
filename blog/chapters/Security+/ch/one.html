Security+ Chapter One Review

Redundancy
Redundancy refers to either a system that is duplicated or a fail over to other systems in the event of a malfunction. Failover refers to the process of a system being reconstructed onto another system when a failure has been detected.
RAID Arrays
RAID 0 is called disk striping. It uses multiple drives that are then mapped together as if it were a single drive. This is done mostly for performance, not for fault tolerances. If one drive fails in a RAID 0 array, the entire logical drive becomes unstable.

RAID 1 is called disk mirroring. Disk mirroring provides 100 percent redundancy as everything is stored on two disk. If one disk fails, another disk continues to operate. The failed disk can then be replaced, and the raid can be regenerated.

RAID 3 is called disk striping with a parity disk. This raid level implements fault tolerence by stripping (like RAID 0) along with a seperate disk that stores parity information. Parity information is a value based on the value of the data stored in each disk location. This system ensures data recovery in the event of a failure.

RAID 5 is called disk stripping with parity. This is the most common RAID configuration. It operates similarly to disk stripping, but the parity data is spread across all of the disks in the array instead of being limited to one like in RAID 3. This requires a minimum of three disks and a max of thirty-two.
Official Book Summary
Risk assessment is the process of evaluation and cataloging the threats, vulnerabilities, and weaknesses that exist in the systems being used. Risk assessment should ensure that all are covered.

Security models begin with an understanding of the business issues that an organization is facing. The following business matters must be evaluated:
  • Policies
  • Standards
  • Guidelines
A good policy design includes scop statements, overview statements, accountability expectations, and exceptions. Each of these aspects of a well-crafted policy helps in setting expectations for everyone in a company. For a policy to be effective, it needs the unequivocal support of senior management and decision makers in an organization.
